# DaoyouCode - 新一代 AI 编程助手

## 项目概述（约200字）

DaoyouCode 是一个基于多 Agent 协作架构的智能编程助手系统，采用先进的 LLM 技术和智能编排策略，为开发者提供代码理解、编写、重构和测试的全方位支持。

**项目信息**
- 技术栈：Python 3.10+ / LLM / LSP / AST / Tree-sitter
- 代码规模：10,000+ 行核心代码，80%+ 测试覆盖率
- 核心特性：6 个专业 Agent、7 种编排策略、34+ 专业工具
- 开源地址：https://github.com/daoyou-zhang/daoyoucode

企业需要对大型代码库进行架构分析和代码审查。DaoyouCode 使用 Code Analyzer Agent 自动分析代码架构，LSP 集成提供类型检查和引用分析，自动生成代码地图识别潜在问题。多专家协作给出改进建议，节省 70% 的人工审查时间。

企业需要重构老旧代码，提升可维护性。Code Analyzer 识别代码坏味道，Refactor Master 提出重构方案，Programmer 执行具体修改，Test Expert 生成测试用例保障质量。系统化的重构流程降低风险，保证代码质量。

### 新人快速上手
新员工需要快速理解复杂项目。系统自动生成项目文档和代码地图，语义搜索快速定位相关代码，交互式问答理解业务逻辑，智能推荐学习路径。缩短 50% 的上手时间，降低培训成本。

### 技术债务管理

## 核心功能（约550字）

### 多 Agent 智能协作
6 个专业 Agent（Sisyphus、Code Analyzer、Programmer、Refactor Master、Test Expert、Librarian）通过 4 种协作模式（Sequential、Parallel、Debate、Main-with-Helpers）智能协作。基于意图识别自动选择 Agent，并行执行提升效率，自动综合多专家意见。

### 深度代码理解
集成 LSP（Language Server Protocol）、AST（抽象语法树）分析和语义代码搜索。支持 Python、JavaScript、TypeScript 等主流语言，提供类型推导、引用分析、基于向量的语义检索。自动识别代码结构和依赖关系。

### 智能编排系统
7 种编排策略（Simple、ReAct、Multi-Agent、Parallel、Conditional、Workflow、Sisyphus）自动选择最优方案。ReAct 模式支持推理-行动循环，条件分支实现动态决策，并行执行提升性能。

### 完整工具链
34+ 专业工具，包含项目理解（文档发现、结构分析、代码地图）、代码搜索（文本、正则、语义、AST）、LSP 工具（诊断、跳转、引用、重命名）、文件操作（读写、批量操作、补丁）、Git 工具（状态、差异、日志）。

### 智能记忆系统
分层存储架构（会话级、用户级、全局级），对话树结构支持多分支，智能压缩和摘要，相关性检索。多轮对话保持上下文，记住用户偏好和习惯，跨会话知识复用。

### Skill 配置系统
14+ 预置技能，灵活的 Prompt 模板，工具和编排器配置，可扩展的技能定义。YAML 配置驱动，Jinja2 模板引擎，动态加载和热更新。

## 技术实现（约550字）

### 系统架构
采用分层架构：CLI 层（命令行接口）→ Executor 层（任务执行）→ Orchestrator 层（智能编排）→ Agent 层（专业智能体）→ Tool 层（工具集）→ LLM 层（大语言模型）。每层职责清晰，松耦合设计。

### 核心技术

**多 Agent 协作**：主从模式通过意图识别自动选择辅助 Agent，使用 asyncio.gather() 并发调用实现并行执行，主 Agent 综合所有专家意见后输出决策。

**LSP 集成**：启动 Language Server 并调用 LSP 功能（诊断、跳转定义、查找引用、智能重命名）。实现连接池复用 Server 实例，自动重启机制处理崩溃，智能路径解析处理虚拟环境。

**语义搜索**：使用 embedding 模型对代码向量化，构建 faiss 索引，通过查询向量进行语义检索。支持自然语言查询，理解代码意图。

**智能编排**：ReAct 模式实现推理-行动循环，LLM 推理后调用工具，将工具结果反馈给 LLM 继续推理，直到任务完成。

**记忆系统**：分层存储，会话级使用内存，用户级使用 SQLite，全局级使用向量数据库。智能加载策略根据任务复杂度动态选择加载范围。

### 性能优化
工具调用优化：同轮去重、结果缓存（TTL: 5分钟）、批量操作、asyncio 并发。Prompt 优化：智能压缩、分层加载、模板复用。超时恢复：逐步增加超时时间、简化 Prompt、切换备用模型。

## 技术难点与解决方案（约450字）

### 多 Agent 协作的结果聚合
**问题**：多个 Agent 并行执行，如何有效聚合结果并解决冲突？  
**解决方案**：设计主从架构，Sisyphus 作为主 Agent 负责综合。标准化 Agent 输出格式（AgentResult），实现结果聚合算法识别共识和分歧，使用 LLM 进行自然语言综合。

### LSP 集成的稳定性
**问题**：Language Server 启动慢、容易崩溃、路径问题。  
**解决方案**：实现 LSP 连接池复用 Server 实例，自动重启机制处理崩溃，智能路径解析处理虚拟环境和多项目，超时保护防止阻塞。LSP 调用成功率 95%+，平均响应时间 < 500ms。

### 大规模代码库的性能
**问题**：大型项目（10,000+ 文件）的代码理解和搜索性能。  
**解决方案**：增量索引只索引变更文件，分层缓存（会话级 + 用户级 + 全局级），智能采样大文件只索引关键部分，并行处理多进程加速索引构建。10,000 文件项目索引时间 < 5 分钟，搜索响应时间 < 1 秒。

### Prompt 工程的可维护性
**问题**：Prompt 分散在代码中，难以维护和优化。  
**解决方案**：设计 Skill 系统，Prompt 独立配置。Jinja2 模板引擎支持变量和逻辑，分层 Prompt 结构（系统 + 用户 + 上下文），版本管理支持 A/B 测试。
