## 🎯 快速开始

### 我是新人，从哪里开始？

1. **第一步**: 阅读 [核心设计文档.md](../核心设计文档.md) (10分钟)
   - 了解系统架构
   - 理解智能体系统
   - 掌握核心概念

2. **第二步**: 浏览 [完整功能清单.md](../完整功能清单.md) (15分钟)
   - 了解有哪些功能
   - 理解功能优先级
   - 找到感兴趣的模块

3. **第三步**: 设置开发环境 [开发指南](zh-CN/DEVELOPMENT.md) (30分钟)
   - 安装依赖
   - 配置环境
   - 运行第一个示例

4. **第四步**: 查看 [项目结构设计.md](../项目结构设计.md) (按需)
   - 了解代码组织
   - 找到对应模块位置
   - 开始编码

## 📝 文档维护

### 文档更新原则

1. **核心设计文档** 是唯一的真理来源
   - 所有重大决策都应该在这里体现
   - 保持简洁，只包含核心概念
   - 定期审查和更新

2. **项目结构设计** 是实现参考
   - 详细的技术细节
   - 代码组织方式
   - 随着实现进度更新

3. **完整功能清单** 是功能索引
   - 所有功能的完整列表
   - 功能来源和实现方式
   - 功能优先级

### 文档更新流程

```
需求变更 → 更新核心设计文档 → 更新相关参考文档 → 代码实现
```
