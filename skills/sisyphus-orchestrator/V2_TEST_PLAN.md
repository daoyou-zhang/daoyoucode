# Sisyphus V2 测试计划

## 🎯 测试目标

验证 V2 版本相比 V1 的改进效果，确保：
1. 回复更自然（不再格式化）
2. 决策更智能（基于场景）
3. 效率更高（减少工具调用）
4. 用户体验更好

## 📋 测试用例

### 用例1：了解项目

**输入**：
```
这个项目是干什么的？
```

**预期行为（V2）**：
- 检查 helper_results 中 code_analyzer 的分析
- 如果信息足够，直接使用（不调用工具）
- 如果信息不足，调用 repo_map 补充
- 回复自然、简洁，不使用格式化模板

**评估指标**：
- [ ] 工具调用次数 ≤ 1
- [ ] 回复无格式化标题（如 "## 📋 任务理解"）
- [ ] 回复简洁（< 200字）
- [ ] 回复自然（像人说话）

### 用例2：定位代码

**输入**：
```
登录功能在哪里实现的？
```

**预期行为（V2）**：
- 检查 helper_results 中是否已定位
- 如果已定位，直接使用
- 如果未定位，调用 semantic_code_search
- 回复直接给出位置，不罗列过程

**评估指标**：
- [ ] 工具调用次数 ≤ 1
- [ ] 回复直接给出文件路径
- [ ] 回复包含简要说明
- [ ] 无冗余信息

### 用例3：修复Bug

**输入**：
```
修复登录Bug
```

**预期行为（V2）**：
- 检查 helper_results 中的分析
- 如果有具体方案，直接采纳
- 如果需要补充，调用 read_file 查看代码
- 回复先说问题，再说方案，最后询问是否执行

**评估指标**：
- [ ] 工具调用次数 ≤ 2
- [ ] 回复结构：问题 → 方案 → 询问
- [ ] 回复自然，无格式化
- [ ] 给出具体的修复步骤

### 用例4：咨询建议

**输入**：
```
chat_assistant.md 有什么优化建议？
```

**预期行为（V2）**：
- 识别为"咨询"（不是"执行"）
- 检查 helper_results 中的建议
- 整合专家意见，给出具体建议
- 不执行修改操作

**评估指标**：
- [ ] 不调用 write_file 或 search_replace
- [ ] 给出3-5条具体建议
- [ ] 建议可执行、有价值
- [ ] 回复自然

### 用例5：执行修改

**输入**：
```
帮我优化 chat_assistant.md
```

**预期行为（V2）**：
- 识别为"执行"（不是"咨询"）
- 基于 helper_results 中的建议
- 使用 search_replace 执行修改
- 回复说明完成了哪些修改

**评估指标**：
- [ ] 调用 search_replace（不是 write_file）
- [ ] 修改精确、有针对性
- [ ] 回复说明修改内容
- [ ] 回复自然

### 用例6：复杂任务

**输入**：
```
重构 agent.py 并添加测试
```

**预期行为（V2）**：
- 识别为复杂任务
- 检查 helper_results 中所有专家的意见
- 整合意见，给出分步方案
- 询问用户是否开始执行

**评估指标**：
- [ ] 工具调用次数 ≤ 3
- [ ] 回复包含分步方案
- [ ] 方案清晰、可执行
- [ ] 询问用户确认

### 用例7：信息不足

**输入**：
```
优化性能
```

**预期行为（V2）**：
- 识别信息不足
- 询问用户具体想优化什么
- 给出几个可能的方向
- 不盲目调用工具

**评估指标**：
- [ ] 工具调用次数 = 0（或 ≤ 1）
- [ ] 回复询问具体需求
- [ ] 给出2-3个可能的方向
- [ ] 回复自然、友好

- **改进**：提升 33%

## 🐛 问题排查

### 问题1：V2 回复仍然格式化

**可能原因**：
- LLM 习惯性使用格式化
- Prompt 中的示例不够明确

**解决方法**：
- 在 Prompt 中增加更多"不好的回复"示例
- 强调"不要使用 ##、###、【】等格式"

### 问题2：V2 工具调用次数没有减少

**可能原因**：
- helper_results 中信息不足
- LLM 没有理解"避免浪费"的原则

**解决方法**：
- 检查 helper_results 的内容
- 在 Prompt 中更强调"先检查 helper_results"

### 问题3：V2 决策不够智能

**可能原因**：
- 场景识别不准确
- 缺少某些场景的处理策略

**解决方法**：
- 补充更多场景示例
- 优化场景识别逻辑
