# 理解项目工作流

## 目标
帮助用户快速理解当前项目的架构、功能和技术栈。

## 关键原则
- 从宏观到微观：先文档、再结构、后代码
- 提炼核心信息，不要罗列细节
- 关注架构和设计思想

## 推荐工具
- `discover_project_docs` - 获取项目文档（README、CONTRIBUTING等）
- `get_repo_structure` - 查看目录组织
- `repo_map` (enable_lsp=true) - 查看代码结构和引用关系

## 注意事项
- 先了解项目背景和目标，再深入代码
- 使用 repo_map 时注意项目规模，大项目可能需要调整 max_tokens
- 提炼核心信息，用自然语言解释，不要简单罗列
