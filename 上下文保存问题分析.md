# 上下文保存问题分析

## 核心问题

### 问题描述
在工作流执行过程中，工具调用的结果（如文件路径）需要在后续步骤中使用，但目前：
1. **没有系统级的上下文保存机制**
2. **只能依赖 LLM 记忆**
3. **容易出错和遗漏**

### 当前解决方案的本质

我们在工作流中做的优化（路径处理规则）实际上是：
- ✅ **通过 Prompt 工程让 LLM 记住路径**
- ✅ **通过明确的指令让 LLM 提取和传递路径**
- ❌ **并没有真正的系统级上下文保存**

## 问题层次分析

### 层次1：工作流层面（已解决 ✅）

**问题**：工作流中硬编码路径

- ✅ 灵活

**缺点**：
- ❌ 依赖 LLM 理解和执行

- ❌ 没有强制性

**问题**：没有工具调用结果的上下文保存机制

#### 2.1 上下文管理器设计

```python
class ExecutionContext:
    """工作流执行上下文"""
    
    def __init__(self):
        self._context = {}
        self._tool_results = []
    
    def set(self, key: str, value: Any):
        """保存变量到上下文"""
        self._context[key] = value
    
    def get(self, key: str, default=None):
        """从上下文获取变量"""
        return self._context.get(key, default)
    
    def save_tool_result(self, tool_name: str, result: Any):
        """保存工具调用结果"""
        self._tool_results.append({
            "tool": tool_name,
            "result": result,
            "timestamp": time.time()
        })
    
    def get_last_result(self, tool_name: str = None):
        """获取最近的工具结果"""
        if tool_name:
            for r in reversed(self._tool_results):
                if r["tool"] == tool_name:
                    return r["result"]
        return self._tool_results[-1]["result"] if self._tool_results else None
    
    def extract_paths(self):
        """自动从工具结果中提取路径"""
        paths = []
        for result in self._tool_results:
            if result["tool"] in ["text_search", "repo_map"]:
                # 自动提取路径
                extracted = self._extract_paths_from_result(result["result"])
                paths.extend(extracted)
        return paths
```

#### 2.2 工具调用集成

```python
class Agent:
    def __init__(self):
        self.context = ExecutionContext()
    
    async def execute_tool(self, tool_name: str, **kwargs):
        """执行工具并保存结果到上下文"""
        
        # 执行工具
        result = await self.tools[tool_name].execute(**kwargs)
        
        # 保存结果到上下文
        self.context.save_tool_result(tool_name, result)
        
        # 自动提取和保存路径
        if tool_name in ["text_search", "repo_map"]:
            paths = self._extract_paths(result)
            if paths:
                self.context.set("last_found_paths", paths)
                if len(paths) == 1:
                    self.context.set("target_file", paths[0])
        
        return result
```

## 当前解决方案的局限性

### 局限1：依赖 LLM 理解

**问题**：
```markdown
步骤1：搜索定位
text_search(query="class BaseAgent", file_pattern="**/*.py")
→ target_file = "backend/daoyoucode/agents/core/agent.py"
```

**依赖**：
- LLM 需要理解 "→ target_file = ..." 的含义
- LLM 需要记住 target_file 的值
- LLM 需要在后续步骤中使用 target_file

**风险**：
- LLM 可能不理解
- LLM 可能忘记
- LLM 可能混淆多个变量

### 局限2：没有强制性

**问题**：
即使工作流中明确说明，LLM 仍然可能：
- 跳过提取步骤
- 忘记使用变量
- 重新硬编码路径

**没有系统级的强制机制**

### 局限3：每个工作流都要重复

**问题**：
每个工作流都需要：
- 说明如何提取路径
- 说明如何保存变量
- 说明如何使用变量

**重复且容易遗漏**

### 局限4：无法处理复杂场景

**问题**：
```markdown
步骤1：搜索文件A
→ file_a = "path/to/a.py"

步骤2：搜索文件B
→ file_b = "path/to/b.py"

步骤3：查找文件A的引用
→ refs_a = [...]

步骤4：查找文件B的引用
→ refs_b = [...]

步骤5：更新文件A的引用
# LLM 需要记住 file_a 和 refs_a
# 容易混淆
```

**多个变量时，LLM 容易混淆**

## 真正的问题所在

### 问题1：系统架构问题 ⭐⭐⭐⭐⭐

**根本原因**：
daoyoucode 的 Agent 执行模型是：
```
LLM → 决策 → 工具调用 → 结果返回给 LLM → LLM 决策 → ...
```

**缺少**：
- 工具调用结果的结构化存储
- 上下文变量的管理
- 自动的信息提取

**这是架构层面的问题，不是工作流层面的问题**

### 问题2：工具结果格式不统一 ⭐⭐⭐⭐

**问题**：
不同工具返回的结果格式不同：
- text_search：返回文本，包含路径
- read_file：返回文件内容
- lsp_find_references：返回引用列表

**没有统一的结果格式，难以自动提取信息**

### 问题3：缺少变量系统 ⭐⭐⭐⭐⭐

**问题**：
工作流中使用的 "变量"（如 target_file）只是：
- Prompt 中的文本
- LLM 需要理解和记忆
- 没有真正的变量系统

**理想状态**：
```python
# 工作流中定义变量
{{target_file}} = text_search(...)

# 后续步骤自动替换
read_file(file_path={{target_file}})
```

```python
# backend/daoyoucode/agents/core/agent.py

class Agent:
    def __init__(self, config):
        self.config = config
        self.context = {}  # 添加上下文
        self.tool_results = []  # 保存工具结果
    
    async def execute_tool(self, tool_name, **kwargs):
        # 执行工具
        result = await self.tools[tool_name].execute(**kwargs)
        
        # 保存结果
        self.tool_results.append({
            "tool": tool_name,
            "result": result
        })
        
        # 简单的自动提取
        if tool_name == "text_search":
            paths = self._extract_paths_from_search(result)
            if paths:
                self.context["last_search_paths"] = paths
        
        return result
    
    def _extract_paths_from_search(self, result):
        """从搜索结果中提取路径"""
        import re
        # 匹配路径模式：path/to/file.py:line
        pattern = r'([^\s:]+\.py):\d+'
        matches = re.findall(pattern, str(result))
        return list(set(matches))
```

**效果**：
- 自动提取和保存路径
- 减少 LLM 负担
- 提高可靠性
