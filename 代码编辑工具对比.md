# 代码编辑工具对比

## 概述

daoyoucode 提供了三种主要的代码编辑工具，各有优势和适用场景。

---

## 工具对比

### 1. search_replace（推荐）⭐⭐⭐⭐⭐

**描述**：使用 SEARCH/REPLACE 模式编辑文件

**优势**：
- ✅ 精确、安全
- ✅ 支持 9 种智能匹配策略
- ✅ 自动处理空白和缩进差异
- ✅ 不需要完整文件内容
- ✅ 执行速度快

**适用场景**：
- 修改函数实现
- 修改类方法
- 修改配置值
- 添加/删除几行代码
- 修改范围 < 20 行

**使用示例**：
```python
search_replace(
    file_path="backend/config.py",
    search="timeout = 30",
    replace="timeout = 60"
)
```

**9 种智能匹配策略**：
1. 简单匹配 - 直接字符串匹配
2. 行首尾 trim - 忽略行首尾空白
3. 块锚定 - 使用首尾行作为锚点
4. 空白归一化 - 忽略空白差异
5. 缩进灵活 - 忽略缩进差异
6. 转义处理 - 处理转义字符
7. 边界 trim - 忽略边界空白
8. 上下文感知 - 基于上下文匹配
9. 多次出现 - 处理多次出现

---

### 2. intelligent_diff_edit（推荐）⭐⭐⭐⭐

**描述**：使用智能 Diff 编辑文件（精确到行，支持模糊匹配）

**优势**：
- ✅ 精确到行
- ✅ 支持模糊匹配（Levenshtein 距离）
- ✅ 自动 LSP 验证
- ✅ 支持流式显示
- ✅ 验证失败自动回滚
- ✅ 不需要完整文件内容

**适用场景**：
- 重构函数
- 修改类结构
- 批量修改相似代码
- 修改范围 20-100 行
- 需要模糊匹配的场景

**使用示例**：
```python
intelligent_diff_edit(
    file_path="backend/agent.py",
    search_block="""
    def execute(self, task):
        # 旧实现
        return result
    """,
    replace_block="""
    async def execute(self, task):
        # 新实现（异步）
        return await result
    """,
    fuzzy_match=True,
    similarity_threshold=0.8
)
```

**特性**：
- 模糊匹配：相似度阈值可调（0.0-1.0）
- 自动验证：使用 LSP 检查语法
- 自动回滚：验证失败时恢复原文件
- 流式显示：实时显示编辑过程

---

### 3. write_file（谨慎使用）⭐⭐⭐

**描述**：写入完整文件内容

**优势**：
- ✅ 适合新建文件
- ✅ 适合完全重写
- ✅ 支持流式显示
- ✅ 自动 LSP 验证

**劣势**：
- ❌ 需要完整文件内容
- ❌ 容易出错（遗漏代码）
- ❌ 不适合小范围修改
- ❌ 效率较低

**适用场景**：
- 创建新文件
- 完全重写文件
- 大规模重构（> 100 行）
- 文件结构完全改变

**使用示例**：
```python
write_file(
    file_path="backend/new_module.py",
    content="""
# 完整的文件内容
import os

class NewClass:
    def __init__(self):
        pass
"""
)
```

## 使用建议

### 优先级排序

1. **search_replace** - 首选（90% 的修改场景）
   - 快速、精确、安全
   - 9 种智能匹配策略
   - 不需要完整文件内容

2. **intelligent_diff_edit** - 次选（需要模糊匹配时）
   - 支持模糊匹配
   - 自动验证和回滚
   - 适合中等范围修改

3. **write_file** - 最后选择（新建文件或大改）
   - 只在必要时使用
   - 容易出错
   - 需要完整文件内容

### 最佳实践

#### 1. 小范围修改（推荐 search_replace）

```python
# ✅ 好的做法
search_replace(
    file_path="config.py",
    search="DEBUG = False",
    replace="DEBUG = True"
)

# ❌ 不好的做法
read_file("config.py")  # 读取完整文件
# 手动修改内容
write_file("config.py", modified_content)  # 重写整个文件
```

#### 2. 中等范围修改（推荐 intelligent_diff_edit）

```python
# ✅ 好的做法
intelligent_diff_edit(
    file_path="agent.py",
    search_block="""
    def process(self, data):
        result = []
        for item in data:
            result.append(item * 2)
        return result
    """,
    replace_block="""
    def process(self, data):
        # 使用列表推导式优化
        return [item * 2 for item in data]
    """,
    fuzzy_match=True
)

# ❌ 不好的做法
write_file("agent.py", entire_file_content)  # 重写整个文件
```

#### 3. 新建文件（使用 write_file）

```python
# ✅ 好的做法
write_file(
    file_path="new_module.py",
    content="""
# 完整的新文件内容
import os

class NewClass:
    pass
"""
)
```

## 常见错误

### 错误1：小改使用 write_file

```python
# ❌ 错误
read_file("config.py")
# 修改一行
write_file("config.py", modified_content)

# ✅ 正确
search_replace(
    file_path="config.py",
    search="old_value",
    replace="new_value"
)
```

### 错误2：不使用智能匹配

```python
# ❌ 错误：精确匹配失败
search_replace(
    file_path="agent.py",
    search="def execute(self, task):",  # 缩进可能不同
    replace="async def execute(self, task):"
)

# ✅ 正确：使用 intelligent_diff_edit 的模糊匹配
intelligent_diff_edit(
    file_path="agent.py",
    search_block="def execute(self, task):",
    replace_block="async def execute(self, task):",
    fuzzy_match=True
)
```

### 错误3：忘记验证

```python
# ❌ 错误：修改后不验证
search_replace(file_path="agent.py", ...)

# ✅ 正确：修改后验证
search_replace(file_path="agent.py", ...)
lsp_diagnostics(file_path="agent.py")  # 检查语法错误
git_diff(file_path="agent.py")  # 查看变更
```

## 总结

### 核心原则

1. **小改用 search_replace** - 快速、精确、安全
2. **中改用 intelligent_diff_edit** - 智能、可靠
3. **大改或新建用 write_file** - 谨慎使用

### 记住

- ✅ 90% 的修改场景应该使用 `search_replace`
- ✅ 需要模糊匹配时使用 `intelligent_diff_edit`
- ✅ 只在新建文件或大规模重构时使用 `write_file`
- ✅ 修改后必须验证（lsp_diagnostics + git_diff）

### 效果对比

使用正确的工具可以：
- 减少 80% 的错误
- 提高 3-5 倍的效率
- 提升代码质量
- 降低维护成本
