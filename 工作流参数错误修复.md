# 工作流参数错误修复

## 问题

在测试时发现错误：
```
TypeError: RepoMapTool.execute() got an unexpected keyword argument 'max_depth'
```

## 原因

工作流中使用了错误的参数 `max_depth`，但 `repo_map` 工具实际上不接受这个参数。

## repo_map 工具的正确参数

根据 `backend/daoyoucode/agents/tools/repomap_tools.py` 的定义：

```python
{
    "repo_path": {
        "type": "string",
        "description": "仓库根目录路径",
        "required": True
    },
    "chat_files": {
        "type": "array",
        "items": {"type": "string"},
        "description": "对话中提到的文件列表（权重×50）",
        "optional": True
    },
    "mentioned_idents": {
        "type": "array",
        "items": {"type": "string"},
        "description": "对话中提到的标识符列表（权重×10）",
        "optional": True
    },
    "max_tokens": {
        "type": "integer",
        "description": "最大token数量（默认3000）",
        "default": 3000,
        "optional": True
    },
    "auto_scale": {
        "type": "boolean",
        "description": "是否自动调整token预算（默认true）",
        "default": True,
        "optional": True
    },
    "enable_lsp": {
        "type": "boolean",
        "description": "是否启用LSP增强（默认true）",
        "default": True,
        "optional": True
    }
}
```

**没有 `max_depth` 参数！**

## 正确的使用方式

### 基本用法
```python
repo_map(repo_path=".")
```

### 获取更多信息
```python
repo_map(
    repo_path=".",
    max_tokens=5000  # 增加token预算
)
```

### 关注特定文件
```python
repo_map(
    repo_path=".",
    chat_files=["backend/agent.py"],  # 提高这些文件的权重
    max_tokens=3000
)
```

### 关注特定标识符
```python
repo_map(
    repo_path=".",
    mentioned_idents=["BaseAgent", "execute"],  # 提高这些标识符的权重
    max_tokens=3000
)
```

### 启用LSP增强
```python
repo_map(
    repo_path=".",
    enable_lsp=true  # 显示类型签名和引用计数
)
```

## 与 get_repo_structure 的区别

### repo_map
- **用途**：获取代码定义和引用关系
- **输出**：类、函数定义，智能排序
- **适合**：了解代码结构、查找定义
- **参数**：repo_path, chat_files, mentioned_idents, max_tokens, enable_lsp

### get_repo_structure
- **用途**：获取目录结构
- **输出**：文件和目录树
- **适合**：了解文件组织
- **参数**：repo_path, max_depth, show_files

## 常见错误

### 错误1：混淆参数 ❌
```python
# 错误：repo_map 不接受 max_depth
repo_map(repo_path=".", max_depth=3)

# 正确：使用 max_tokens
repo_map(repo_path=".", max_tokens=5000)
```

### 错误2：混淆工具 ❌
```python
# 如果想要目录结构，应该用 get_repo_structure
get_repo_structure(repo_path=".", max_depth=3)

# 如果想要代码定义，应该用 repo_map
repo_map(repo_path=".")
```
