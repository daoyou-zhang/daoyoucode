# 工作流路径传递设计

## 核心问题

工作流中不能写死路径，因为：
1. 不同项目的目录结构不同
2. 运行目录可能不同（项目根目录 vs backend 目录）
3. 文件位置可能变化
4. 需要支持通用性

## 当前问题

### 问题1：路径硬编码
```markdown
# ❌ 错误做法
write_file(file_path="daoyoucode/agents/core/config.py", ...)
```

### 问题2：工具结果没有传递
```markdown
# ❌ 错误做法
1. text_search(query="agent.py")  # 找到文件
2. read_file(file_path="???")  # 不知道用什么路径
```

### 问题3：LLM 需要自己推理路径
- LLM 需要记住之前工具返回的路径
- 容易出错
- 效率低

## 解决方案

### 方案1：在工作流中明确路径传递规则 ⭐⭐⭐⭐⭐

**核心思想**：工作流明确告诉 LLM 如何获取和使用路径

#### 示例：拆分大类工作流

```markdown
#### 步骤1：定位目标文件

**目标**：找到要拆分的文件的准确路径

**工具序列**：
1. 使用 text_search 查找文件
   ```
   text_search(
       query="class BaseAgent",
       file_pattern="**/*.py"
   )
   ```
   
   **输出示例**：
   ```
   找到 1 个匹配:
   - backend/daoyoucode/agents/core/agent.py:50
   ```
   
   **提取路径**：从结果中提取文件路径
   → `target_file = "backend/daoyoucode/agents/core/agent.py"`

2. 验证文件存在
   ```
   read_file(file_path=target_file)
   ```
   
   **如果成功**：确认路径正确
   **如果失败**：重新搜索或调整路径

**重要**：将 `target_file` 路径记住，后续步骤都使用这个路径

#### 步骤2：分析文件结构

**使用上一步获取的路径**：

```
get_file_symbols(file_path=target_file)
```

**输出**：类和方法列表

#### 步骤3：确定新文件路径

**基于目标文件路径，构造新文件路径**：

```
如果 target_file = "backend/daoyoucode/agents/core/agent.py"
那么：
- config_file = "backend/daoyoucode/agents/core/config.py"
- llm_caller_file = "backend/daoyoucode/agents/core/llm_caller.py"
```

**规则**：
- 新文件与原文件在同一目录
- 只改变文件名，不改变目录路径

#### 步骤4：创建新文件

**使用步骤3构造的路径**：

```
write_file(
    file_path=config_file,
    content="..."
)
```

#### 步骤5：验证新文件

**使用相同的路径**：

```
lsp_diagnostics(file_path=config_file)
```

#### 步骤6：更新原文件

**使用步骤1获取的路径**：

```
search_replace(
    file_path=target_file,
    search="...",
    replace="..."
)
```
```

### 方案2：使用变量占位符（需要系统支持）⭐⭐⭐

**需要系统支持**：context 变量传递

```markdown
#### 步骤1：定位文件
```
text_search(query="class BaseAgent")
→ 将结果路径保存到 {{target_file}}
```

#### 步骤2：读取文件
```
read_file(file_path={{target_file}})
```

#### 步骤3：分析结构
```
get_file_symbols(file_path={{target_file}})
```
```

**优点**：
- 清晰
- 不容易出错

**缺点**：
- 需要系统支持变量
- 当前 daoyoucode 可能不支持

### 方案3：在 Prompt 中强调路径传递 ⭐⭐⭐⭐

**核心思想**：通过清晰的指令让 LLM 理解路径传递

```markdown
## 路径处理规则 ⚠️⚠️⚠️

### 规则1：永远不要硬编码路径

❌ 错误：
```
write_file(file_path="backend/daoyoucode/agents/core/config.py", ...)
```

✅ 正确：
```
1. 先搜索找到目标文件路径
2. 基于目标文件路径构造新文件路径
3. 使用构造的路径
```

### 规则2：路径传递流程

```
步骤1：搜索定位
text_search(query="class BaseAgent") 
→ 得到路径：backend/daoyoucode/agents/core/agent.py

步骤2：记住路径
在后续步骤中，始终使用这个路径：
- target_file = "backend/daoyoucode/agents/core/agent.py"
- target_dir = "backend/daoyoucode/agents/core"

步骤3：构造新路径
基于 target_dir 构造新文件路径：
- config_file = target_dir + "/config.py"
- llm_caller_file = target_dir + "/llm_caller.py"

步骤4：使用路径
所有后续工具调用都使用这些路径
```

### 规则3：路径验证

每次使用路径前，先验证：
```
read_file(file_path=target_file)
→ 如果成功：路径正确
→ 如果失败：重新搜索
```

### 规则4：路径提取

从工具结果中提取路径：
```
text_search 结果：
  backend/daoyoucode/agents/core/agent.py:50: class BaseAgent

提取路径：
  file_path = "backend/daoyoucode/agents/core/agent.py"
  
提取目录：
  dir_path = "backend/daoyoucode/agents/core"
```
```

## 推荐方案：方案1 + 方案3

结合两种方案的优点：

### 在工作流中：

1. **明确的步骤顺序**
2. **清晰的路径传递规则**
3. **具体的示例**
4. **错误处理**

### 示例工作流

```markdown
## 拆分大类工作流

### 阶段1：定位和分析

#### 步骤1.1：搜索目标文件
```
工具：text_search
参数：
  - query: "class BaseAgent"（用户提到的类名）
  - file_pattern: "**/*.py"

输出示例：
  backend/daoyoucode/agents/core/agent.py:50: class BaseAgent

⚠️ 重要：从输出中提取文件路径
  → target_file = "backend/daoyoucode/agents/core/agent.py"
  → target_dir = "backend/daoyoucode/agents/core"
```

#### 步骤1.2：验证文件路径
```
工具：read_file
参数：
  - file_path: target_file（使用步骤1.1提取的路径）

目的：
  - 确认文件存在
  - 确认路径正确
  - 读取文件内容
```

#### 步骤1.3：分析文件结构
```
工具：get_file_symbols
参数：
  - file_path: target_file（使用相同的路径）

输出：类和方法列表
```

### 阶段2：创建新文件

#### 步骤2.1：构造新文件路径
```
基于 target_dir，构造新文件路径：

config_file = target_dir + "/config.py"
  → "backend/daoyoucode/agents/core/config.py"

llm_caller_file = target_dir + "/llm_caller.py"
  → "backend/daoyoucode/agents/core/llm_caller.py"

⚠️ 规则：
  - 新文件与原文件在同一目录
  - 只改变文件名
  - 不要硬编码目录路径
```

#### 步骤2.2：创建配置文件
```
工具：write_file
参数：
  - file_path: config_file（使用步骤2.1构造的路径）
  - content: "..."

⚠️ 不要写成：
  file_path: "backend/daoyoucode/agents/core/config.py"
```

#### 步骤2.3：验证新文件
```
工具：lsp_diagnostics
参数：
  - file_path: config_file（使用相同的路径）
```

### 阶段3：更新原文件

#### 步骤3.1：更新导入
```
工具：search_replace
参数：
  - file_path: target_file（使用阶段1提取的路径）
  - search: "from dataclasses import dataclass"
  - replace: "from dataclasses import dataclass\nfrom .config import AgentConfig"
```

#### 步骤3.2：验证修改
```
工具：lsp_diagnostics
参数：
  - file_path: target_file（使用相同的路径）
```

### 阶段4：查找引用

#### 步骤4.1：查找所有引用
```
工具：lsp_find_references
参数：
  - file_path: target_file（使用阶段1提取的路径）
  - line: 类定义行号
  - character: 0

输出：所有引用该类的文件列表
```

#### 步骤4.2：更新每个引用
```
对每个引用文件：

工具：search_replace
参数：
  - file_path: 引用文件路径（从步骤4.1的输出中获取）
  - search: "..."
  - replace: "..."
```

## 路径传递的关键点

### 1. 搜索是起点
```
text_search → 获取准确路径 → 后续所有步骤使用这个路径
```

### 2. 路径构造规则
```
原文件：/path/to/agent.py
新文件：/path/to/config.py（同目录，不同文件名）
```

### 3. 路径验证
```
每次使用路径前，先用 read_file 验证
```

### 4. 路径一致性
```
同一个文件，在所有步骤中使用相同的路径
```

### 路径处理规则 ⚠️

2. **先搜索，后使用**

**工具**：text_search

**工具**：read_file

### 步骤3：使用路径

**工具**：[任何工具]
**参数**：file_path = target_file

### 步骤4：构造新路径

**规则**：基于 target_file 构造
**示例**：new_file = target_dir + "/new_file.py"

### 步骤5：使用新路径

**工具**：write_file
**参数**：file_path = new_file
```

## 总结

### 核心原则

1. ✅ **搜索定位** - 使用 text_search 找到准确路径
2. ✅ **提取路径** - 从搜索结果中提取路径
3. ✅ **验证路径** - 使用 read_file 验证
4. ✅ **传递路径** - 后续步骤使用相同路径
5. ✅ **构造路径** - 基于已知路径构造新路径
6. ✅ **不要硬编码** - 永远不要写死路径

### 实施方案

1. **在工作流中明确路径传递规则**
2. **提供清晰的示例**
3. **强调路径验证**
4. **使用一致的变量名**（target_file, config_file, etc.）

### 预期效果

- ✅ 工作流可以应用于任何项目
- ✅ 路径自动适应不同的目录结构
- ✅ 减少路径错误
- ✅ 提高工作流的通用性
