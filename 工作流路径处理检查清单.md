# 工作流路径处理检查清单

## 用途

在创建或更新工作流时，使用此检查清单确保路径处理正确。

## 核心检查项

### ✅ 1. 是否避免了硬编码路径？

**检查点**：
- [ ] 工作流中没有写死的文件路径（如 `backend/daoyoucode/agents/core/agent.py`）
- [ ] 所有路径都通过搜索工具动态获取
- [ ] 示例中使用变量名而不是具体路径

**错误示例**：
```markdown
使用工具：read_file
参数：
  - file_path: "backend/daoyoucode/agents/core/agent.py"
```

**正确示例**：
```markdown
使用工具：read_file
参数：
  - file_path: target_file（使用步骤0提取的路径）
```

### ✅ 2. 是否包含路径定位步骤？

**检查点**：
- [ ] 工作流开始时有明确的"定位文件"步骤
- [ ] 使用 text_search 或 repo_map 查找文件
- [ ] 从搜索结果中提取路径
- [ ] 验证路径是否正确

**标准模板**：
```markdown
#### 步骤1：定位目标文件

1. 搜索文件
   text_search(query="用户提到的特征", file_pattern="**/*.py")
   
2. 提取路径
   从结果中提取：
   → target_file = "搜索结果中的路径"
   → target_dir = "文件所在目录"
   
3. 验证路径
   read_file(file_path=target_file)
```

### ✅ 3. 是否使用一致的变量命名？

**检查点**：
- [ ] 使用标准变量名（target_file, target_dir, config_file 等）
- [ ] 变量名在整个工作流中保持一致
- [ ] 每个变量都有明确的说明

**标准变量名**：
- `target_file` - 目标文件的完整路径
- `target_dir` - 目标文件所在目录
- `config_file` - 配置文件路径
- `new_file` - 新创建的文件路径
- `ref_file_1`, `ref_file_2` - 引用文件路径

### ✅ 4. 是否包含路径验证步骤？

**检查点**：
- [ ] 使用 LSP 工具前先验证文件存在
- [ ] 创建新文件后验证创建成功
- [ ] 每个关键步骤都有验证

**标准模板**：
```markdown
步骤1：确认文件存在
read_file(file_path=target_file)

步骤2：使用 LSP 工具
lsp_diagnostics(file_path=target_file)
```

### ✅ 5. 是否说明了路径相对性？

**检查点**：
- [ ] 说明路径相对于当前工作目录
- [ ] 提醒不要假设路径格式
- [ ] 强调先搜索再使用

**标准说明**：
```markdown
⚠️ 路径说明：
- 路径相对于当前工作目录
- 可能包含或不包含 backend 前缀
- 不要假设路径格式
- 必须先搜索定位
```

### ✅ 6. 是否包含路径传递说明？

**检查点**：
- [ ] 说明如何从搜索结果提取路径
- [ ] 说明如何基于已知路径构造新路径
- [ ] 说明如何在步骤间传递路径

**标准模板**：
```markdown
步骤1：提取路径
从搜索结果中提取：
→ target_file = "搜索结果中的路径"
→ target_dir = "文件所在目录"

步骤2：构造新路径
基于 target_dir 构造：
→ config_file = target_dir + "/config.py"

步骤3：使用路径
write_file(file_path=config_file, ...)
```

### ✅ 7. 是否包含常见错误说明？

**检查点**：
- [ ] 列出硬编码路径的错误
- [ ] 列出跳过搜索步骤的错误
- [ ] 列出不验证路径的错误
- [ ] 提供正确做法的对比

**标准模板**：
```markdown
## 常见错误

❌ 错误1：硬编码路径
```
write_file(file_path="backend/daoyoucode/agents/core/config.py", ...)
```

✅ 正确做法：
```
1. 先搜索定位
2. 提取路径
3. 使用路径变量
```
```

### ✅ 8. 是否包含完整示例？

**检查点**：
- [ ] 提供从搜索到使用的完整示例
- [ ] 示例中使用路径变量
- [ ] 示例中包含验证步骤
- [ ] 示例易于理解和遵循

**标准模板**：
```markdown
## 完整示例

# 步骤1：定位文件
text_search(query="class BaseAgent", file_pattern="**/*.py")
→ target_file = "backend/daoyoucode/agents/core/agent.py"
→ target_dir = "backend/daoyoucode/agents/core"

# 步骤2：验证路径
read_file(file_path=target_file)

# 步骤3：构造新路径
config_file = target_dir + "/config.py"

# 步骤4：创建文件
write_file(file_path=config_file, content="...")

# 步骤5：验证新文件
lsp_diagnostics(file_path=config_file)
```

## 工作流模板

### 基础模板

```markdown
# [工作流名称]

## 路径处理规则 ⚠️

### 核心原则：永远不要硬编码路径

❌ 错误做法：
```
write_file(file_path="backend/path/to/file.py", ...)
```

✅ 正确做法：
```
1. 使用 text_search 定位文件
2. 从搜索结果提取路径
3. 使用 read_file 验证路径
4. 基于已知路径构造新路径
5. 所有后续步骤使用提取的路径
```

## 执行步骤

### 步骤1：定位目标文件（必须第一步）

**工具**：text_search
**参数**：
  - query: "用户提到的特征"
  - file_pattern: "**/*.py"

**输出**：文件路径

**提取**：
- target_file = "搜索结果中的路径"
- target_dir = "文件所在目录"

**验证**：
read_file(file_path=target_file)

### 步骤2：[具体操作]

**使用路径**：target_file（从步骤1提取）

**工具**：[工具名称]
**参数**：
  - file_path: target_file

### 步骤3：创建新文件（如果需要）

**构造路径**：
new_file = target_dir + "/new_file.py"

**工具**：write_file
**参数**：
  - file_path: new_file
  - content: "..."

**验证**：
lsp_diagnostics(file_path=new_file)

## 常见错误

❌ **错误1：硬编码路径**
```

```

```

```

✅ **正确做法**：
```
先使用 text_search 查找

## 常见问题

### Q1: 什么时候可以使用具体路径？

A: 只在说明路径格式时可以使用，但必须标注为示例：

```markdown
⚠️ 示例（不要直接使用）：
如果 target_dir = "backend/daoyoucode/agents/core"
则 config_file = "backend/daoyoucode/agents/core/config.py"
```

### Q2: 如何处理多个文件？

A: 为每个文件使用独立的变量：

```markdown
# 定位第一个文件
text_search(query="class A", file_pattern="**/*.py")
→ file_a = "搜索结果中的路径"

# 定位第二个文件
text_search(query="class B", file_pattern="**/*.py")
→ file_b = "搜索结果中的路径"
```

### Q3: 如何处理引用文件？

A: 从 lsp_find_references 结果中提取：

```markdown
# 查找引用
lsp_find_references(file_path=target_file, line=50, character=0)

输出：
- path/to/ref1.py:25
- path/to/ref2.py:10

# 提取引用路径
ref_file_1 = "path/to/ref1.py"
ref_file_2 = "path/to/ref2.py"

# 使用引用路径
search_replace(file_path=ref_file_1, ...)
```

### Q4: 如何处理相对路径？

A: 始终使用相对于当前工作目录的路径，并说明：

```markdown
⚠️ 路径说明：
- 路径相对于当前工作目录
- 如果在 backend 目录下运行，使用：daoyoucode/agents/core/agent.py
- 如果在项目根目录运行，使用：backend/daoyoucode/agents/core/agent.py
```

## 总结

### 核心要点

1. **永远不要硬编码路径** - 这是最重要的原则
2. **先搜索，再使用** - 使用 text_search 定位文件
3. **提取路径** - 从搜索结果中提取路径
4. **验证路径** - 使用 read_file 验证
5. **构造新路径** - 基于已知路径构造
6. **使用变量** - 使用路径变量，不要重复硬编码
7. **验证每步** - 每步都要验证，失败就回退

### 记住这个流程

```
搜索 → 提取 → 验证 → 构造 → 使用 → 验证
```

这个流程适用于所有工作流！
