# 路径处理快速参考

## 🎯 核心原则（3条）

### 1. 路径相对于当前工作目录
```python
# 在 backend 目录下运行
✅ daoyoucode/agents/core/agent.py
❌ backend/daoyoucode/agents/core/agent.py

# 在项目根目录运行
✅ backend/daoyoucode/agents/core/agent.py
❌ daoyoucode/agents/core/agent.py
```

### 2. 永远不要猜测路径
```python
❌ 错误：根据类名推测路径
lsp_diagnostics(file_path="backend/daoyoucode/agents/core/agent.py")

✅ 正确：先搜索，再使用
text_search(query="class BaseAgent", file_pattern="**/*.py")
→ 找到：daoyoucode/agents/core/agent.py
lsp_diagnostics(file_path="daoyoucode/agents/core/agent.py")
```

### 3. LSP 工具前必须确认文件存在
```python
❌ 错误：直接使用 LSP
lsp_diagnostics(file_path="推测的路径")
→ 失败：文件不存在

✅ 正确：先确认，再使用
read_file(file_path="路径")  # 确认存在
lsp_diagnostics(file_path="路径")  # 使用 LSP
```

## 📋 标准流程

### 流程1：查找文件
```
1. 搜索文件
   text_search(query="class BaseAgent", file_pattern="**/*.py")
   或
   repo_map(repo_path=".")

2. 提取路径
   从结果中提取：daoyoucode/agents/core/agent.py

3. 确认存在
   read_file(file_path="daoyoucode/agents/core/agent.py")
   → 成功：路径正确
```

### 流程2：使用 LSP 工具
```
1. 确认文件存在（必须）
   read_file(file_path="daoyoucode/agents/core/agent.py")

2. 使用 LSP 工具
   lsp_diagnostics(file_path="daoyoucode/agents/core/agent.py")
   或
   lsp_find_references(
       file_path="daoyoucode/agents/core/agent.py",
       line=100,
       character=0
   )
```

### 流程3：创建新文件
```
1. 确定目标路径
   - 在 backend 目录下：daoyoucode/agents/core/new_file.py
   - 在项目根目录：backend/daoyoucode/agents/core/new_file.py

2. 创建文件
   write_file(
       file_path="daoyoucode/agents/core/new_file.py",
       content="..."
   )

3. 验证创建成功
   read_file(file_path="daoyoucode/agents/core/new_file.py")
```

## ⚠️ 常见错误

### 错误1：硬编码 backend 前缀
```python
❌ lsp_diagnostics(file_path="backend/daoyoucode/agents/core/agent.py")
✅ lsp_diagnostics(file_path="daoyoucode/agents/core/agent.py")  # 在 backend 目录下
```

### 错误2：推测路径
```python
❌ 看到类名 BaseAgent，推测路径是 backend/daoyoucode/agents/core/agent.py
✅ 使用 text_search 查找实际路径
```

### 错误3：不确认文件存在
```python
❌ 直接使用 lsp_diagnostics(file_path="推测的路径")
✅ 先 read_file 确认，再使用 lsp_diagnostics
```

### 错误4：使用绝对路径
```python
❌ file_path="D:/daoyouspace/daoyoucode/backend/daoyoucode/agents/core/agent.py"
✅ file_path="daoyoucode/agents/core/agent.py"  # 相对路径
```

## 🔧 工具使用

### 查找文件的工具
```python
# 推荐：repo_map（最快）
repo_map(repo_path=".")
→ 返回所有类、函数的定义和位置

# 备选：text_search（精确）
text_search(query="class BaseAgent", file_pattern="**/*.py")
→ 返回匹配的文件和行号

# 备选：grep_search（模式匹配）
grep_search(query="class\\s+BaseAgent", include_pattern="**/*.py")
→ 返回匹配的文件和上下文
```

### 确认文件的工具
```python
# 唯一选择：read_file
read_file(file_path="路径")
→ 成功：文件存在，路径正确
→ 失败：文件不存在或路径错误
```

### LSP 工具（需要准确路径）
```python
# 诊断错误
lsp_diagnostics(file_path="确认的路径")

# 查找引用
lsp_find_references(
    file_path="确认的路径",
    line=100,
    character=0
)

# 查找符号
lsp_symbols(file_path="确认的路径")

# 查找定义
lsp_find_definition(
    file_path="确认的路径",
    line=100,
    character=0
)
```


## 🚀 快速检查清单

在使用 LSP 工具前，检查：
- [ ] 是否使用了相对路径（相对于当前工作目录）？
- [ ] 是否先搜索了文件位置（text_search 或 repo_map）？
- [ ] 是否确认了文件存在（read_file）？
- [ ] 路径中是否包含了不必要的 backend 前缀？
- [ ] 是否根据类名或模块名推测了路径？

如果以上任何一项回答"否"，请先修正再使用 LSP 工具。
