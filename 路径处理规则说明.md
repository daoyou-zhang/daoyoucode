# 路径处理规则说明

## 问题背景

在测试 daoyoucode 时，发现 LSP 工具报错：
```
文件不存在: backend\daoyoucode\agents\core\agent.py
```

## 根本原因

**路径相对性问题**：工作流中使用的路径示例是绝对路径或包含 `backend` 前缀，但实际运行时：
- 如果在 `backend` 目录下运行 daoyoucode，路径应该是：`daoyoucode/agents/core/agent.py`
- 如果在项目根目录运行，路径应该是：`backend/daoyoucode/agents/core/agent.py`

## 核心原则

### 原则1：使用相对于当前工作目录的路径 ⭐⭐⭐

```python
# 如果在 backend 目录下运行
✅ 正确：daoyoucode/agents/core/agent.py
❌ 错误：backend/daoyoucode/agents/core/agent.py

# 如果在项目根目录运行
✅ 正确：backend/daoyoucode/agents/core/agent.py
❌ 错误：daoyoucode/agents/core/agent.py
```

### 原则2：永远不要猜测路径 ⭐⭐⭐

```python
# 错误做法
❌ 根据类名推测：BaseAgent → backend/daoyoucode/agents/core/agent.py
❌ 直接使用：lsp_diagnostics(file_path="backend/daoyoucode/agents/core/agent.py")

# 正确做法
✅ 先搜索：text_search(query="class BaseAgent", file_pattern="**/*.py")
✅ 确认存在：read_file(file_path="搜索到的路径")
✅ 再使用 LSP：lsp_diagnostics(file_path="确认的路径")
```

### 原则3：LSP 工具前必须确认文件存在 ⭐⭐⭐

```python
# 错误做法
❌ 直接调用 LSP 工具
lsp_diagnostics(file_path="推测的路径")
→ 如果路径错误，报错：文件不存在

# 正确做法
✅ 先确认文件存在
read_file(file_path="路径")
→ 如果成功，说明路径正确

✅ 再调用 LSP 工具
lsp_diagnostics(file_path="确认的路径")
```

## 工具使用流程

### 流程1：查找文件位置

```
步骤1：使用 repo_map 或 text_search 查找
repo_map(repo_path=".")
或
text_search(query="class BaseAgent", file_pattern="**/*.py")

步骤2：从结果中提取文件路径
结果：找到 daoyoucode/agents/core/agent.py

步骤3：确认文件存在
read_file(file_path="daoyoucode/agents/core/agent.py")
→ 成功：路径正确
→ 失败：尝试其他路径
```

### 流程2：使用 LSP 工具

```
步骤1：确认文件存在（必须）
read_file(file_path="daoyoucode/agents/core/agent.py")

步骤2：使用 LSP 工具
lsp_diagnostics(file_path="daoyoucode/agents/core/agent.py")
或
lsp_find_references(
    file_path="daoyoucode/agents/core/agent.py",
    line=100,
    character=0
)
```

### 流程3：创建新文件

```
步骤1：确定目标路径
- 如果在 backend 目录下运行：daoyoucode/agents/core/new_file.py
- 如果在项目根目录运行：backend/daoyoucode/agents/core/new_file.py

步骤2：创建文件
write_file(
    file_path="daoyoucode/agents/core/new_file.py",
    content="..."
)

步骤3：验证文件创建成功
read_file(file_path="daoyoucode/agents/core/new_file.py")
```

## 总结

### 核心要点

1. **路径相对性** - 路径相对于当前工作目录，不是项目根目录
2. **不要猜测** - 永远不要根据类名或模块名推测路径
3. **先确认后使用** - 使用 LSP 工具前，必须先确认文件存在
4. **使用搜索工具** - 使用 text_search 或 repo_map 查找文件位置
