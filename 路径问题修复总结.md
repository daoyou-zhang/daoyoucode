# 路径问题修复总结

## 问题描述

在测试 daoyoucode 拆分 agent.py 任务时，出现路径错误：
```
文件不存在: backend\daoyoucode\agents\core\agent.py
```

## 根本原因

工作流中的路径示例使用了硬编码的 `backend/` 前缀，但实际运行时：
- 如果在 `backend` 目录下运行 daoyoucode → 路径应该是 `daoyoucode/agents/core/agent.py`
- 如果在项目根目录运行 → 路径应该是 `backend/daoyoucode/agents/core/agent.py`

## 核心问题

### 问题1：路径相对性不明确
工作流示例中使用绝对路径或包含 `backend` 前缀，导致 AI 直接复制使用，但实际路径取决于运行位置。

### 问题2：缺少路径确认步骤
工作流没有强调在使用 LSP 工具前必须先确认文件存在，导致 AI 直接使用推测的路径。

### 问题3：示例路径硬编码
工作流中的示例使用了具体的项目路径（如 `backend/daoyoucode/...`），AI 会直接复制这些路径。

## 修复方案

### 方案1：添加路径处理规则 ⭐⭐⭐

在关键工作流中添加路径处理规则说明：

```markdown
**⚠️ 路径处理规则**：
- 路径应该相对于当前工作目录
- 如果在 backend 目录下运行，使用：daoyoucode/agents/core/agent.py
- 如果在项目根目录运行，使用：backend/daoyoucode/agents/core/agent.py
- 不要猜测路径，先使用 text_search 或 repo_map 查找
- LSP 工具需要准确的文件路径
```

### 方案2：强制路径确认步骤 ⭐⭐⭐

在使用 LSP 工具前，强制要求确认文件存在：

```markdown
步骤1：确认文件存在（必须）
read_file(file_path="要检查的文件")
→ 成功：路径正确
→ 失败：尝试其他路径

步骤2：使用 LSP 工具
lsp_diagnostics(file_path="确认的路径")
```

### 方案3：去除硬编码路径示例 ⭐⭐

将工作流中的具体路径示例替换为通用说明：

```markdown
# 修改前
file_path="backend/daoyoucode/agents/core/agent.py"

# 修改后
file_path="搜索到的文件路径"

⚠️ 注意：使用相对于当前工作目录的路径
```
