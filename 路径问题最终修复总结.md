# 路径问题最终修复总结

## 修复时间
2026-03-08

## 核心问题

工作流中存在硬编码路径，导致：
1. 无法应用于不同的项目
2. 在不同目录运行时路径错误
3. LLM 容易猜测错误的路径
4. 工具调用失败

## 解决方案

### 核心原则：动态路径获取和传递

```
步骤0：搜索定位（必须第一步）
  ↓
步骤1：提取路径
  ↓
步骤2：验证路径
  ↓
步骤3：构造新路径
  ↓
步骤4：使用路径变量
```

### 关键改进

#### 1. 添加"步骤0：定位目标文件"

在"模式3：拆分大类"的最开始添加了必须执行的步骤0：

```markdown
#### 步骤0：定位目标文件（必须第一步）⭐⭐⭐

工具序列：
1. 搜索目标文件
   text_search(query="用户提到的类名", file_pattern="**/*.py")
   
2. 提取路径信息
   → target_file = "搜索结果中的路径"
   → target_dir = "文件所在目录"
   
3. 验证文件存在
   read_file(file_path=target_file)
```

#### 2. 添加"路径处理规则"部分

在"模式3"开头添加了详细的路径处理规则：

```markdown
## 路径处理规则 ⚠️⚠️⚠️

### 核心原则：永远不要硬编码路径

❌ 错误做法：
write_file(file_path="backend/daoyoucode/agents/core/config.py", ...)

✅ 正确做法：
1. 使用 text_search 定位文件
2. 从搜索结果提取路径
3. 使用 read_file 验证路径
4. 基于已知路径构造新路径
5. 所有后续步骤使用提取的路径
```

#### 3. 重写所有步骤使用路径变量

所有步骤现在都使用从步骤0提取的路径变量：

**步骤1：分析类的职责**
```markdown
⚠️ 使用步骤0获取的 target_file

工具：
1. read_file(file_path=target_file)
2. get_file_symbols(file_path=target_file)
```

**步骤2：创建配置文件**
```markdown
⚠️ 路径构造：基于步骤0获取的 target_dir

构造新文件路径：
config_file = target_dir + "/config.py"

工具：write_file
参数：
  - file_path: config_file（使用构造的路径变量）
```

**步骤3：创建第一个新类**
```markdown
⚠️ 路径构造：基于步骤0获取的 target_dir

构造新文件路径：
llm_caller_file = target_dir + "/llm_caller.py"

工具：write_file
参数：
  - file_path: llm_caller_file（使用构造的路径变量）
```

**步骤4：更新原类使用新类**
```markdown
⚠️ 使用步骤0获取的 target_file

工具：search_replace（多次调用）

search_replace(
    file_path=target_file,  # 使用步骤0的路径变量
    search="...",
    replace="..."
)
```

**步骤5：查找并更新引用**
```markdown
⚠️ 使用步骤0获取的 target_file

工具：lsp_find_references

lsp_find_references(
    file_path=target_file,  # 使用步骤0的路径变量
    line=类定义行号,
    character=0
)

对每个引用文件：
提取引用文件路径：
→ ref_file_1 = "从结果中提取的路径"

search_replace(
    file_path=ref_file_1,  # 使用提取的路径
    search="...",
    replace="..."
)
```

#### 4. 更新完整示例

```markdown
第0步：定位文件（必须）
text_search(query="class BaseAgent", file_pattern="**/*.py")
→ target_file = "backend/daoyoucode/agents/core/agent.py"
→ target_dir = "backend/daoyoucode/agents/core"

第1轮：拆分配置类
1. 构造路径：config_file = target_dir + "/config.py"

3. 更新 agent.py 导入（使用 target_file）

第2轮：拆分 LLM 调用
1. 构造路径：llm_caller_file = target_dir + "/llm_caller.py"
2. 创建 llm_caller.py
3. 更新 agent.py（使用 target_file）
4. 查找并更新所有引用（使用 target_file）
5. 验证
```

#### 5. 添加新的常见错误

```markdown
❌ 错误1：硬编码路径（最严重）

```

将"不要硬编码路径"放在第一位，强调其重要性。

## 路径传递模式

### 模式总结

```
定位阶段：
  text_search → 提取路径 → 验证路径
  ↓
  target_file, target_dir

创建阶段：
  构造新路径 → 创建文件 → 验证文件
  ↓
  config_file, llm_caller_file, ...

更新阶段：
  使用 target_file → 修改原文件 → 验证修改
  ↓
  查找引用 → 提取引用路径 → 更新引用
  ↓
  ref_file_1, ref_file_2, ...
```

### 变量命名规范

- `target_file` - 要重构的目标文件路径
- `target_dir` - 目标文件所在目录
- `config_file` - 配置文件路径
- `llm_caller_file` - LLM 调用器文件路径
- `tool_manager_file` - 工具管理器文件路径
- `memory_manager_file` - 记忆管理器文件路径
- `ref_file_1`, `ref_file_2`, ... - 引用文件路径
